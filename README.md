Process Manager Ultimate
Обзор
Process Manager Ultimate — мощное приложение для управления процессами в Windows с графическим интерфейсом на WPF. Программа позволяет отслеживать запущенные процессы, изменять их приоритеты, управлять привязкой к ядрам процессора (CPU affinity), просматривать дерево процессов, анализировать использование памяти и CPU в реальном времени.
Приложение разработано на C# с использованием .NET 8, MVVM, LiveCharts для графиков и WMI для сбора расширенной информации.

Возможности
Список процессов с подробной информацией:

PID, имя, приоритет, потребление памяти, количество потоков, время CPU, состояние (отвечает/не отвечает)

Управление приоритетами (6 уровней: от Низкого до Реального времени)

Привязка к ядрам процессора (CPU affinity) — выбор конкретных ядер для выполнения процесса

Массовое выделение процессов и массовое изменение приоритета/affinity

Дерево процессов — отображение иерархии (родитель-потомок)

Потоки процесса — просмотр всех потоков выбранного процесса с их приоритетами и состоянием

График загрузки CPU в реальном времени (обновление каждую секунду)

Топ-10 процессов по памяти — быстрый доступ к наиболее ресурсоёмким приложениям

Индикация приоритетов цветом для наглядности

Русский интерфейс и русские названия приоритетов

Технологии
.NET 8 (Windows Desktop)

WPF (Windows Presentation Foundation)

MVVM с реализацией INotifyPropertyChanged

LiveCharts.Wpf — визуализация графика CPU

System.Management (WMI) — получение дерева процессов

System.Diagnostics — работа с процессами и потоками

PerformanceCounter — мониторинг загрузки процессора

Требования
Windows 10 / 11 (64‑bit)

Установленный .NET 8 Runtime (приложение может требовать его, но при сборке можно включить самодостаточный режим)

Права администратора (для изменения приоритетов и affinity)

Сборка и запуск
1. Клонирование репозитория
bash
git clone https://github.com/ваш_аккаунт/ProcessManagerUltimate.git
cd ProcessManagerUltimate
2. Открытие проекта
Откройте файл решения ProcessManagerUltimate.sln в Visual Studio 2022 или более поздней версии.

3. Восстановление пакетов
NuGet-пакеты восстановятся автоматически при сборке. При необходимости выполните в консоли:

bash
dotnet restore
4. Сборка
В Visual Studio: выберите конфигурацию Release, затем Сборка → Собрать решение.

Или через dotnet:

bash
dotnet build -c Release
5. Запуск
В Visual Studio: нажмите F5 (Debug) или Ctrl+F5 (без отладки).

Или выполните:

bash
dotnet run --project ProcessManagerUltimate/ProcessManagerUltimate.csproj
Важно: Для корректной работы с некоторыми процессами (особенно системными) требуется запуск от имени администратора. Если вы столкнулись с ошибками при изменении приоритетов или affinity, запустите приложение с правами администратора.

Использование
Основное окно
Список процессов отображается в левой части.

Выберите процесс для просмотра деталей — информация появится в правой панели и вкладке "Потоки".

Для массового выделения используйте кнопки "Выделить всё", "Снять выделение" или вручную ставьте галочки в первом столбце.

Массовое изменение приоритета/affinity применяется ко всем выделенным процессам.

Управление affinity
В столбце "Ядра" можно вручную ввести номера ядер через запятую (например, 0,2,4). Изменение применяется автоматически после потери фокуса ячейки.

В правой панели под "Выбранный процесс" вы можете визуально выбрать ядра через чекбоксы (каждое ядро отмечено галочкой) — изменения применяются к выбранному процессу.

График CPU
Отображает среднюю загрузку процессора по всем ядрам.

Обновляется каждую секунду, сохраняет последние 60 значений.

Топ-10 по памяти
Автоматически обновляется и показывает 10 процессов, потребляющих больше всего оперативной памяти.
